# Requirements — v2.12 Advanced Quantization

## Goals & Success Criteria

1. Add FP8 support (E4M3/E5M2) for next-gen transformer efficiency
2. Implement CUDA kernels for all quantization operations
3. Production-grade calibration utilities with histogram-based methods
4. Comprehensive benchmark framework comparing precision modes
5. QAT (Quantization-Aware Training) integration patterns

## Functional Requirements

### Must Have

- [ ] **FP8-01:** FP8 type definitions (E4M3, E5M2) with conversion to/from FP32
- [ ] **FP8-02:** CUDA kernel for FP8 matrix multiplication (with FP32 accumulation)
- [ ] **FP8-03:** FP8 activation functions (ReLU, GELU) with numerical safety
- [ ] **CAL-01:** Histogram-based calibration with percentile selection
- [ ] **CAL-02:** MinMax and MSE calibration methods
- [ ] **CAL-03:** Per-channel calibration with channel-wise scales
- [ ] **KERN-01:** CUDA kernels for INT8/FP16 quantization (move from CPU)
- [ ] **KERN-02:** Vectorized load/store for quantization (128-bit wide)

### Should Have

- [ ] **QAT-01:** FakeQuantize op pattern for training integration
- [ ] **QAT-02:** Straight-through estimator (STE) gradient approximation
- [ ] **MIX-01:** Automatic mixed precision (AMP) manager with precision selection
- [ ] **MIX-02:** Layer-wise precision assignment based on sensitivity analysis
- [ ] **BENCH-01:** Quantization benchmark suite (FP32 vs FP16 vs INT8 vs FP8)
- [ ] **BENCH-02:** Accuracy comparison tool with reference baseline

### Could Have

- [ ] **AUTO-01:** Automatic precision selection based on loss sensitivity
- [ ] **AUTO-02:** Profile-guided precision optimization

## Non-Functional Requirements

### Performance

- FP8 matmul: >4x speedup vs FP32, >2x vs FP16
- Quantization overhead: <1ms for 1B parameter tensor
- Calibration time: <10s for 100GB dataset sample

### Accuracy

- FP8 relative error: <0.1% vs FP32 for typical workloads
- INT8 accuracy: Within 0.5% of FP32 for transformer layers
- Calibration preserves >99% accuracy vs floating-point

### Reliability

- No NaN/Inf propagation in FP8 kernels
- Graceful handling of outlier values in calibration
- Deterministic results with fixed random seeds

## Constraints & Assumptions

- CUDA 20+ required for FP8 intrinsics
- cuBLASLt available for FP8 GEMM acceleration
- No breaking changes to existing quantize_tensor.hpp API
- Maintain backward compatibility with v2.6 inference module

## Out of Scope

- PTQ (Post-Training Quantization) beyond calibration
- Dynamic quantization for variable-length sequences
- INT4/INT2 quantization
- Vendor-specific optimizations (Tensor Cores not directly exposed)

## Dependencies

- Existing: quantize_tensor.hpp, quantize_ops.hpp
- Existing: linalg/matmul.hpp for base matmul
- Existing: neural/activations.hpp for activation functions
- New: CUDA 20 FP8 intrinsics (or software fallback)
