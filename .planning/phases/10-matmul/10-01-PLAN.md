## Key Design Decisions

### 1. Row-Wise Split Strategy

Each GPU computes a contiguous block of output rows:
- GPU i computes rows `[i * m / n, (i + 1) * m / n)` of output C
- Weight matrix B is replicated on all GPUs
- Uses `DistributedAllGather` from Phase 8 to gather partial results

```
Layout for 3 GPUs, m=9, n=6:
  A: [9 x k] (full input)
  B: [k x 6] (full weights, replicated)
  C: [9 x 6] (full output)

GPU 0: C[0:3, :] = A[0:3, :] @ B
GPU 1: C[3:6, :] = A[3:6, :] @ B
GPU 2: C[6:9, :] = A[6:9, :] @ B

After local matmul, each GPU has its slice. All-gather reconstructs full C on each GPU.
```

### 2. Per-Device cuBLAS Handles

Each GPU needs its own `cublasHandle_t`. The `DistributedMatmul` class manages a vector of handles.

### 3. Single-GPU Fallback

When `device_count <= 1`:
- Delegate directly to `cuda::neural::matmul`
- No multi-GPU primitives called
- No peer access checks needed

### 4. ParallelismStrategy Enum

Per ARCHITECTURE.md, use `ParallelismStrategy` enum for future expansion:
- `DataParallel`: Row-partition input A (current implementation)
- `TensorParallel`: Column-partition weights (deferred to v1.2)
- `PipelineParallel`: Layer pipelining (deferred to v1.2)

## Implementation Plan

### Step 1: Create Header (`include/cuda/distributed/matmul.h`)

```cpp
namespace cuda::distributed {

enum class ParallelismStrategy {
    DataParallel,   // Row-partition A (this phase)
    TensorParallel, // Column-partition W (v1.2)
    PipelineParallel // Layer pipeline (v1.2)
};

struct DistributedMatmulOptions {
    ParallelismStrategy strategy = ParallelismStrategy::DataParallel;
    float alpha = 1.0f;
    float beta = 0.0f;
    bool trans_a = false;
    bool trans_b = false;
};

class DistributedMatmul {
public:
    // Main entry point: distribute across all available GPUs
    static void matmul(
        const float* A,  // Full input [m x k]
        const float* B,  // Full weights [k x n]
        float* C,        // Full output [m x n]
        int m, int n, int k,
        DistributedMatmulOptions options = {}
    );

    // Async version with explicit stream
    static void matmul_async(
        const float* A, const float* B, float* C,
        int m, int n, int k,
        cudaStream_t stream,
        DistributedMatmulOptions options = {}
    );

    // Single-GPU fallback (exposed for testing)
    static void matmul_single_gpu(
        const float* A, const float* B, float* C,
        int m, int n, int k,
        DistributedMatmulOptions options = {}
    );

    static bool needs_multi_gpu();
};

}  // namespace cuda::distributed
```

### Step 2: Create Implementation (`src/cuda/distributed/matmul.cu`)

**Algorithm:**

```cpp
void DistributedMatmul::matmul_async(...) {
    auto& mesh = cuda::mesh::DeviceMesh::instance();
    mesh.initialize();

    int n = mesh.device_count();

    // Single-GPU fallback (MGPU-13)
    if (n <= 1) {
        matmul_single_gpu(A, B, C, m, n, k, options);
        return;
    }

    int rank;
    CUDA_CHECK(cudaGetDevice(&rank));

    // Calculate partition for this GPU
    int rows_per_gpu = m / n;
    int start_row = rank * rows_per_gpu;
    int local_m = (rank == n - 1) ? (m - start_row) : rows_per_gpu;

    // Allocate local output buffer
    float* local_C;
    CUDA_CHECK(cudaMalloc(&local_C, local_m * n * sizeof(float)));

    // Switch to this GPU and compute local matmul
    {
        cuda::mesh::ScopedDevice guard(rank);

        // Get per-device cuBLAS handle
        cublasHandle_t handle = get_cublas_handle_for_device(rank);

        MatmulOptions opts;
        opts.handle = handle;
        opts.alpha = options.alpha;
        opts.beta = options.beta;
        opts.trans_a = options.trans_a;
        opts.trans_b = options.trans_b;

        // Compute C[start_row:start_row+local_m, :] = A[start_row:start_row+local_m, :] @ B
        cuda::neural::matmul(
            A + start_row * k,  // Offset A to local rows
            B,
            local_C,
            local_m, n, k,
            opts
        );
    }

    // All-gather to reconstruct full output (MGPU-12)
    // Each GPU's local_C has local_m rows, output needs m rows
    DistributedAllGather::all_gather_async(
        local_C,
        C + start_row * n,  // Offset to correct position
        local_m * n,
        0,  // stream
        cudaDataType::CUDA_R_32F
    );

    // Synchronize
    CUDA_CHECK(cudaDeviceSynchronize());

    // Cleanup
    CUDA_CHECK(cudaFree(local_C));
}

// Per-device cuBLAS handle cache
cublasHandle_t get_cublas_handle_for_device(int device) {
    static std::vector<cublasHandle_t> handles;
    static std::vector<int> handle_devices;

    // Check cache
    for (size_t i = 0; i < handle_devices.size(); ++i) {
        if (handle_devices[i] == device) {
            return handles[i];
        }
    }

    // Create new handle
    cuda::mesh::ScopedDevice guard(device);
    cublasHandle_t handle;
    CUBLAS_CHECK(cublasCreate(&handle));
    handles.push_back(handle);
    handle_devices.push_back(device);
    return handle;
}
```

```cpp
TEST_F(DistributedMatmulTest, OutputMatchesReference) {
    constexpr float kTolerance = 1e-3f;

    // Setup: allocate test matrices
    const int m = 1024, n = 512, k = 256;
    cuda::memory::Buffer<float> h_A(m * k);
    cuda::memory::Buffer<float> h_B(k * n);
    cuda::memory::Buffer<float> h_C_single(m * n);
    cuda::memory::Buffer<float> h_C_multi(m * n);

    // Fill with random data
    fill_random(h_A.data(), m * k);
    fill_random(h_B.data(), k * n);

    // Single-GPU reference
    cuda::neural::matmul(h_A.data(), h_B.data(), h_C_single.data(), m, n, k);

    // Multi-GPU result
    cuda::distributed::DistributedMatmul::matmul(
        h_A.data(), h_B.data(), h_C_multi.data(), m, n, k);

    // Compare
    std::vector<float> single_result(m * n), multi_result(m * n);
    h_C_single.copy_to(single_result.data(), m * n);
    h_C_multi.copy_to(multi_result.data(), m * n);

    for (int i = 0; i < m * n; ++i) {
        EXPECT_NEAR(single_result[i], multi_result[i], kTolerance)
            << "Mismatch at index " << i;
    }
}
```

## Success Criteria (Observable Behaviors)

1. **Row-wise split multi-GPU matmul produces numerically correct results**
   - On 2+ GPU system, `DistributedMatmul::matmul` output matches `cuda::neural::matmul` reference within 1e-3 absolute error for random matrices
   - Each GPU computes its partition of rows correctly

2. **Single-GPU fallback bypasses all multi-GPU code paths**
   - On single-GPU system, `DistributedMatmul::matmul` delegates to `cuda::neural::matmul`
   - No `DistributedAllGather` calls, no peer access checks
   - `CUDA_VISIBLE_DEVICES=""` test passes

3. **Multi-GPU matmul accepts configurable number of GPUs**
   - Works with any device count from 1 to max
   - Gracefully handles non-peer-capable pairs (via all-gather fallback)
   - Row distribution correct for non-divisible m (last GPU gets remainder)

## Notes

- **cuBLAS handle management**: Per-device handles are cached to avoid repeated `cublasCreate`/`cublasDestroy` calls
- **Memory allocation**: Local output buffer is allocated per-call (temporary). For performance, callers could pre-allocate and pass as options
- **Async execution**: `matmul_async` is non-blocking but `matmul` (without `_async`) calls `cudaDeviceSynchronize` for simplicity
- **All-gather correctness**: The all-gather in this implementation uses a simplified pattern where each GPU gathers into its portion. Full all-gather where all GPUs receive all portions is deferred
